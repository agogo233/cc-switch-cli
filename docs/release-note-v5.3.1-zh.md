# CC Switch CLI v5.3.1

> 聚焦于 Codex 官方认证安全、代理热切换和新版数据库兼容性的补丁版本。

**[English Version →](release-note-v5.3.1-en.md)**

---

## 更新概览

**发布日期**：2026-04-12

- **Codex 官方 Provider 安全性**：编辑 Codex 官方 provider 时会保留已有认证快照，不再把它重建成第三方中转配置。
- **运行中代理热切换**：在 takeover 已启用时，切换 provider 会立即更新当前代理目标，不再依赖手动重启 proxy 才能生效。
- **数据库 Schema v8 兼容**：后端现在可以接受更新后的 schema v8 数据库，并支持本地旧数据库按 v6 -> v7 -> v8 路径分阶段升级。
- **定价数据一致性**：迁移场景和新建数据库场景都会使用修正后的定价数据，避免导入库和新库表现不一致。

## 已修复

- **Issue #102**：修复 Codex 官方 provider 路径错误回写 `base_url`、并导致官方认证快照丢失的问题。
- **Issue #95**：修复 takeover 运行中切换 provider 仍需手动重启 proxy 才能生效的问题。
- **Issue #106**：修复导入新版数据库时报“数据库版本过新（8）”的阻塞问题，补齐支持版本和迁移链路。

## 说明

- 这是一次有意控制范围的补丁发布，不是新一轮功能扩展。
- 现有数据库仍按原地升级处理；新创建的数据库会直接使用当前版本要求的最新 schema。
